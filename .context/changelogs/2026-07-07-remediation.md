# 2026-07-07 — Remediation: plan vs. what actually landed

> **Status**: Historical · **Created**: 2026-07-08 · **Source**: doc/plan/2026-07-07-remediation.md + `git log master..fix/remediation-2026-07-07`

The 2026-07-07 review surfaced an install-breaking naming drift, a world-writable data API, an incoherent billing model, and broken auth/workspaces/UI code. The remediation plan (`doc/plan/2026-07-07-remediation.md`) was executed on branch `fix/remediation-2026-07-07` — 44 commits, `232167a..4ff8cb5`. Net-new features (email, notifications, onboarding, storage, social login) were explicitly out of scope. This entry records what each phase **actually** delivered, cross-checked against the code, not the plan's intent.

Note on sequencing: the plan file itself was committed mid-branch (`800acbb`), after the Phase 0–3 work and a security redesign that went beyond it had already landed.

## Bottom line

Every phase landed commits; nothing was skipped outright. The security-critical work (Phases 2–4) shipped a design **stronger** than planned (Lambda-mediated writes + Cognito group-per-workspace instead of the interim IAM-only scheme). The one materially unfinished item is Phase 1: CI is wired but red-by-design until `amplify_outputs.json` is stubbed/generated in the workflow and the typecheck baseline is paid down. Deploy-dependent verifications (sandbox curl proof, Stripe CLI webhook trigger) are not evidenced in the repo. Residual defects listed above are tracked as tech debt, not regressions introduced by this branch.
